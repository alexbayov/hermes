# Hermes upgrade project

Этот раздел — постоянное поле работ по апгрейду Hermes.

Цель: новый Devin должен открыть эти файлы, быстро понять текущий статус, выбрать следующий безопасный шаг и оформить задачу/PR без повторного аудита с нуля.

## Как пользоваться

1. Начни с `docs/project/status.md`.
2. Выбери первый незакрытый `P0`, затем `P1`.
3. Открой карточку в `docs/project/backlog.md`.
4. Проверь зависимости, файлы и acceptance criteria.
5. Сделай маленький PR только по одной карточке.
6. После merge обнови:
   - статус карточки в `backlog.md`;
   - чекбокс в `status.md`;
   - `Last updated`;
   - ссылку на PR/commit.

## Файлы проекта

| Файл | Назначение |
| --- | --- |
| `status.md` | Краткая доска прогресса: что готово, что следующее |
| `backlog.md` | Детальные карточки задач с критериями приемки |
| `roles.md` | Роли для будущих Devin-сессий и зона ответственности |
| `handoff.md` | Инструкция новому Devin: как войти в контекст и не сломать профиль |
| `architecture.md` | Целевая архитектура control layer |
| `decision-log.md` | Журнал принятых проектных решений |

## Статусы

| Статус | Значение |
| --- | --- |
| `TODO` | Не начато |
| `READY` | Можно брать без дополнительных решений |
| `IN_PROGRESS` | Уже есть активная работа/PR |
| `PARTIAL` | Часть подготовлена, но acceptance criteria ещё не закрыты |
| `BLOCKED` | Нужен ответ Alex или внешняя зависимость |
| `DONE` | Реализовано и проверено |
| `DEFERRED` | Осознанно отложено |

## Приоритеты

| Приоритет | Смысл |
| --- | --- |
| `P0` | Безопасность профиля и быстрые защитные фиксы |
| `P1` | Control layer: hard-stop, state, recovery, anti-carousel |
| `P2` | Observability и behavior exams |
| `P3` | Browser/WebBridge/регистрационные workflow |
| `P4` | Desktop GUI adoption after control layer is safe |

## Главный принцип

Не развивать WebBridge, регистрации, billing, browser automation и Desktop GUI как основной интерфейс, пока `P1` control layer не закрыт.
