# Hermes — Self‑Improvement / Autonomous Loop Architecture  Target framework: skill = YAML frontmatter + markdown body, SQLite session DB, cron scheduler, subagent delegation.  The core design decision that makes everything else clean:  > *There is exactly ONE execution loop. "Skills" never call other skills.* > The meta‑skills (sequencer, diagnostician, memory) are *hooks/middleware* that subscribe to lifecycle events emitted by that single loop. Task‑skills are *leaf capabilities* (a deterministic script + an LLM fallback). This kills "nested skill hell" because there is no nesting — there is one loop and a flat set of subscribers.  ---  ## 0. File structure  ``` hermes/ ├── core/ │   ├── loop.py            # the ONE execution loop + event bus │   ├── events.py          # lifecycle event enum + Hook protocol │   ├── loopguard.py       # anti-loop: action hashing + cycle/stall detection │   ├── observation.py     # compact observation builder (interactive elements) │   ├── workspace.py       # draft workspace lifecycle (create/cleanup) │   ├── journal.py         # SQLite journal API (per-draft + global) │   ├── skillgen.py        # draft -> SKILL.md + script extraction │   ├── registry.py        # skill loader (parses frontmatter), hook ordering │   └── runner.py          # deterministic-script-first, LLM-fallback executor ├── skills/ │   ├── _meta/             # the 3 orchestration layers (hooks) │   │   ├── sequencer/SKILL.md │   │   ├── diagnostician/SKILL.md │   │   └── memory/SKILL.md │   └── web_register/      # an EXAMPLE learned task-skill │       ├── SKILL.md │       ├── script.py      # deterministic happy path │       └── templates/ │           └── form_map.yaml ├── drafts/ │   └── {task_id}/         # auto-created at task start │       ├── journal.db     # per-task SQLite journal (isolated) │       ├── work/          # scratch files, screenshots, downloads │       └── candidate/     # skill being assembled before promotion ├── db/ │   └── hermes.db          # global session + learning DB └── cron/     └── schedule.yaml      # cron scheduler entries ```  ---  ## 1. ANTI‑LOOP infrastructure  Four mechanisms, all enforced by `LoopGuard` + `runner`:  1. *Action fingerprint* — `sha1(tool + normalized_args)`, ring buffer of last N. 2. *Stop conditions* — declarative `max_steps` + `success` predicate per task. 3. *Compact observation* — interactive elements list, not raw DOM. 4. *Deterministic‑first execution* — run the script; only call the LLM on `UnexpectedState`.  ### 1.1 Action fingerprint + cycle detection  ```python # core/loopguard.py import hashlib, json from collections import deque, Counter  VOLATILE_KEYS = {"timestamp", "nonce", "request_id", "csrf", "_ts"}  def normalize_args(args: dict) -> dict:     """Drop volatile fields, sort, lowercase string values that are URLs/ids."""     clean = {k: v for k, v in args.items() if k not in VOLATILE_KEYS}     return dict(sorted(clean.items()))  def action_hash(tool: str, args: dict) -> str:     payload = json.dumps([tool, normalize_args(args)], sort_keys=True, default=str)     return hashlib.sha1(payload.encode()).hexdigest()[:16]  class LoopGuard:     def __init__(self, window=8, repeat_limit=3, cycle_len=3, cycle_repeats=2):         self.window = window         self.repeat_limit = repeat_limit      # same exact action N times -> loop         self.cycle_len = cycle_len            # detect A-B-A-B... patterns         self.cycle_repeats = cycle_repeats         self.hashes = deque(maxlen=window)         self.last_obs_hash = None         self.no_progress_steps = 0      def check(self, tool, args, obs_hash) -> str | None:         h = action_hash(tool, args)          # (a) identical action spammed         if Counter(self.hashes)[h] + 1 >= self.repeat_limit:             return f"REPEAT: action {h} repeated {self.repeat_limit}x"          # (b) oscillation: last cycle_len*cycle_repeats hashes form a repeating cycle         seq = list(self.hashes) + [h]         if len(seq) >= self.cycle_len * self.cycle_repeats:             block = seq[-self.cycle_len:]             reps = sum(                 seq[-(i+1)*self.cycle_len:-i*self.cycle_len or None] == block                 for i in range(self.cycle_repeats)             )             if reps >= self.cycle_repeats:                 return f"CYCLE: pattern {block} repeating"          # (c) no observable progress (state hash unchanged after an action)         if obs_hash == self.last_obs_hash:             self.no_progress_steps += 1             if self.no_progress_steps >= self.repeat_limit:                 return "STALL: environment state unchanged after repeated actions"         else:             self.no_progress_steps = 0          self.hashes.append(h)         self.last_obs_hash = obs_hash         return None        # None == proceed ```  `check()` returns `None` to proceed or a reason string. A non‑None result is raised as a *stuck signal*, which triggers the `on_stuck` event (→ diagnostician, §3).

*PART 1/3 (cont.)*  ### 1.2 Declarative stop conditions  Each task carries a contract — stored in the `task` row and on the skill frontmatter:  ```yaml # part of a task spec / skill frontmatter stop:   max_steps: 25   max_seconds: 240   success:                      # ALL must hold     - kind: url_matches       pattern: "/welcome|/dashboard"     - kind: element_present       label: "Log out" ```  ```python # core/loop.py  (predicate evaluation) def success_met(success_spec, obs) -> bool:     for cond in success_spec:         if cond["kind"] == "url_matches" and not re.search(cond["pattern"], obs.url):             return False         if cond["kind"] == "element_present" and \            not any(e["label"] == cond["label"] for e in obs.elements):             return False         if cond["kind"] == "text_present" and cond["text"] not in obs.text:             return False     return True ```  ### 1.3 Compact observation (no 50k DOM)  ```python # core/observation.py from dataclasses import dataclass, field import hashlib, json  @dataclass class Observation:     url: str     text: str = ""                       # short visible text only     elements: list = field(default_factory=list)   # interactive elements      def hash(self) -> str:         # only the actionable surface determines "did the state change"         key = json.dumps([self.url, [(e["ref"], e["value"]) for e in self.elements]],                          sort_keys=True)         return hashlib.sha1(key.encode()).hexdigest()[:16]  def build_observation(page, cap=40) -> Observation:     els = []     for i, node in enumerate(page.query_interactive()):   # buttons/inputs/links/selects         els.append({             "ref":   f"e{i}",                 # stable handle the LLM can reference             "role":  node.role,               # button|textbox|checkbox|link|select             "label": node.accessible_name[:60],             "value": (node.value or "")[:60], # CURRENT value -> lets guard see progress             "state": node.state,              # enabled|disabled|checked         })         if len(els) >= cap:             break     return Observation(url=page.url, text=page.visible_text[:800], elements=els) ```  The LLM sees ~40 elements like `e7 textbox "Email" value="" enabled`, not raw HTML. `value` is included so both the model and the LoopGuard's `obs_hash` can tell whether anything actually changed.  ### 1.4 Deterministic‑first runner  ```python # core/runner.py class UnexpectedState(Exception): ...  def run_step(task, obs, bus):     skill = task.active_skill     # 1) try the deterministic script if the skill has one and preconditions match     if skill and skill.has_script and skill.preconditions_ok(obs):         try:             return skill.script.run_one(obs)        # returns ToolCall         except UnexpectedState as e:             bus.emit("on_unexpected", task, reason=str(e))             # fall through to LLM     # 2) LLM fallback — only reached on first-time or anomaly     return llm_decide(task, obs, tools=task.tools) ```  Routine work (fill known fields → submit) runs as code at ~0 token cost; the model is invoked only when reality diverges from the script.

*PART 2/3*  ## 2. Self‑learning skill formation (draft → skill → cleanup)  ### 2.1 Draft workspace lifecycle  ```python # core/workspace.py import os, shutil, sqlite3, uuid, time  DRAFTS = "drafts"  def open_draft(goal: str) -> dict:     task_id = f"{int(time.time())}_{uuid.uuid4().hex[:6]}"     root = os.path.join(DRAFTS, task_id)     os.makedirs(os.path.join(root, "work"))     os.makedirs(os.path.join(root, "candidate"))     journal = os.path.join(root, "journal.db")     init_journal(journal)                  # see schema §2.3     return {"task_id": task_id, "root": root, "journal": journal, "goal": goal}  def cleanup_draft(draft, keep_artifacts_to=None):     if keep_artifacts_to:                  # copy anything worth keeping first         shutil.copytree(os.path.join(draft["root"], "candidate"),                         keep_artifacts_to, dirs_exist_ok=True)     shutil.rmtree(draft["root"])           # auto-clean temp workspace ```  `open_draft` runs at task start; `cleanup_draft` runs after skill extraction (or on hard failure, optionally retaining `work/` for postmortem).  ### 2.2 Skill extraction (draft → SKILL.md + script)  After success, take the *journaled action trace*, keep only `status='ok'` steps on the winning path, generalize literal args into template variables, and emit a skill.  ```python # core/skillgen.py def extract_skill(draft, global_db):     steps = winning_path(draft["journal"])          # ok steps, dedup retries     params = infer_parameters(steps)                # emails/urls/names -> {{vars}}     name = propose_name(draft["goal"])              # e.g. "web_register"      cand = os.path.join(draft["root"], "candidate")     write_script(cand, steps, params)               # script.py: deterministic replay     write_frontmatter_md(cand, name, draft, params) # SKILL.md     write_templates(cand, steps)                    # templates/form_map.yaml      if validate_skill(cand):                        # dry-run replay in sandbox         promote(cand, dst=f"skills/{name}")         # atomic move into registry         register_skill(global_db, name, source_task=draft["task_id"], version=1)     cleanup_draft(draft) ```  `infer_parameters` is the only "smart" step — it diffs the concrete values against the goal text and the env to decide what is a constant vs. a runtime input (email, password, site URL become `{{params}}`; CSS refs/selectors stay as constants or get re‑resolved by label at replay time).  ### 2.3 Generated artifacts (real examples)  *`skills/web_register/SKILL.md`* ```markdown --- name: web_register kind: task version: 1 source_task: 1781206500_a1b2c3 description: Register a new account on a standard email/password signup form. params:   - {name: site_url,  type: url,    required: true}   - {name: email,     type: string, required: true}   - {name: password,  type: secret, required: true} preconditions:   - {kind: element_present, role: textbox, label_like: "email"} stop:   max_steps: 15   success:     - {kind: text_present, text: "verify your email"} script: script.py ---  ## When to use A site with a conventional signup form (email, password, confirm, submit).  ## Happy path 1. Open `{{site_url}}`, find the signup form. 2. Fill email -> password -> confirm using `templates/form_map.yaml`. 3. Submit; expect an email-verification screen.  ## Fallback If fields can't be matched by label, hand control to the LLM step (runner already does this on `UnexpectedState`). ```  *`skills/web_register/script.py`* ```python from hermes.core.runner import UnexpectedState import yaml, os  FORM_MAP = yaml.safe_load(open(os.path.join(os.path.dirname(__file__),                                             "templates/form_map.yaml")))  def run_one(obs, params):     for field in FORM_MAP["fields"]:                       # ordered, deterministic         el = match_by_label(obs.elements, field["label_like"], field["role"])         if el is None:             raise UnexpectedState(f"field '{field['label_like']}' not found")         if el["value"]:                                    # already filled -> skip (idempotent)             continue         return {"tool": "fill", "ref": el["ref"], "text": params[field["param"]]}     submit = match_by_label(obs.elements, "sign up|create account", "button")     if submit is None:         raise UnexpectedState("submit button not found")     return {"tool": "click", "ref": submit["ref"]} ```  ---  ## 3. Multi‑layer skill orchestration (the 3 meta‑layers)  *Composition rule:* the meta‑skills are *hooks on lifecycle events*, ordered by `priority`. They never invoke each other and never invoke task‑skills. The loop emits events; subscribers react. This is the entire anti‑"nested‑skill‑hell" mechanism.  ### 3.1 Lifecycle events  ```python # core/events.py EVENTS = [     "on_plan",        # task start: build/refresh the step plan     "before_step",    # right before an action is chosen     "after_step",     # action executed + observation in hand     "on_unexpected",  # deterministic script hit reality mismatch     "on_stuck",       # LoopGuard tripped (repeat/cycle/stall)     "on_success",     # stop.success satisfied     "on_failure",     # max_steps/seconds exceeded or fatal error ]  class Hook:                       # every _meta skill implements a subset     priority: int = 100     subscribes: list[str] = []     def handle(self, event, ctx): ... ```  ### 3.2 The three layers  ``` | Layer                | Skill         | Subscribes to           | Job                                                          | |----------------------|---------------|-------------------------|--------------------------------------------------------------| | Sequential discipline| sequencer     | on_plan, before_step    | Maintain ordered plan; step k only after k-1 done.           | | Problem diagnosis    | diagnostician | on_stuck, on_unexpected  | Turn a stuck signal into 2-4 concrete options; pick/escalate.| | Memory / logging     | memory        | after_step, on_success, on_failure | Write every step to the journal; mark outcomes.    | ```  *`skills/_meta/sequencer/SKILL.md`* ```markdown --- name: sequencer kind: meta priority: 10 subscribes: [on_plan, before_step] description: Keeps execution on an ordered plan; blocks out-of-order or skipped steps. --- On `on_plan`: write `ctx.plan` = ordered list of subgoals. On `before_step`: expose `ctx.current_subgoal`; if the proposed action doesn't serve it, veto (return `ctx.redirect`) so the chosen step advances the chain, not a side quest. ```  *`skills/_meta/diagnostician/SKILL.md`* ```markdown --- name: diagnostician kind: meta priority: 20 subscribes: [on_stuck, on_unexpected] description: When stuck, diagnose root cause and emit concrete options instead of retrying blindly. --- On `on_stuck`/`on_unexpected`: 1. Read last K journal rows (failed + their observations). 2. Classify: selector_drift | auth_wall | rate_limit | missing_precondition | genuine_dead_end. 3. Produce options (re-resolve element by label / wait+retry once / switch strategy /    escalate to subagent / ask human). Set `ctx.next_strategy`. Never repeat the tripped action. ```  *`skills/_meta/memory/SKILL.md`* ```markdown --- name: memory kind: meta priority: 5 subscribes: [after_step, on_success, on_failure] description: Persists every step + outcome to the SQLite learning journal. --- On `after_step`: insert a `step` row (tool, args_hash, status, obs_hash, latency). On `on_success`: mark task ok; flag the winning path for skill extraction. On `on_failure`: mark task failed; keep trace for diagnosis/postmortem. ```  ### 3.3 Registry + ordering  ```python # core/registry.py def load_hooks(meta_dir="skills/_meta"):     hooks = [parse_frontmatter(p) for p in glob(f"{meta_dir}/*/SKILL.md")]     return sorted(hooks, key=lambda h: h.priority)     # low priority = runs first  def dispatch(bus_event, ctx, hooks):     for h in hooks:         if bus_event in h.subscribes:             h.handle(bus_event, ctx)                    # mutates ctx, no return chaining ```  Memory (5) logs before sequencer (10) re‑plans before diagnostician (20) intervenes — a deterministic, debuggable order. Add a fourth concern later (e.g., a `budget` guard) = drop one folder in `_meta/`, no rewiring.

*PART 3/3*  ## 4. SQLite schema — the learning journal  Two scopes: *per‑draft* `journal.db` (isolated, fast, disposable) and *global* `hermes.db` (durable session + learning store). Same `step`/`task` shape so promotion is a copy.  ```sql -- ===== global db/hermes.db =====  CREATE TABLE task (     task_id      TEXT PRIMARY KEY,     goal         TEXT NOT NULL,     status       TEXT NOT NULL DEFAULT 'running',  -- running|ok|failed|aborted     skill_used   TEXT,                              -- NULL if solved ad-hoc     steps_taken  INTEGER DEFAULT 0,     created_at   REAL,     finished_at  REAL );  CREATE TABLE step (     id           INTEGER PRIMARY KEY AUTOINCREMENT,     task_id      TEXT NOT NULL REFERENCES task(task_id),     idx          INTEGER NOT NULL,                  -- step number within task     subgoal      TEXT,                              -- from sequencer     tool         TEXT NOT NULL,     args_json    TEXT NOT NULL,     args_hash    TEXT NOT NULL,                     -- LoopGuard fingerprint     obs_hash     TEXT,                              -- state hash AFTER action     status       TEXT NOT NULL,                     -- ok|error|unexpected|vetoed     error        TEXT,     latency_ms   INTEGER,     on_win_path  INTEGER DEFAULT 0,                 -- set 1 by skillgen     created_at   REAL ); CREATE INDEX ix_step_task ON step(task_id, idx); CREATE INDEX ix_step_hash ON step(args_hash);  CREATE TABLE skill (     name         TEXT PRIMARY KEY,     version      INTEGER NOT NULL,     kind         TEXT NOT NULL,                     -- task|meta     source_task  TEXT REFERENCES task(task_id),     path         TEXT NOT NULL,     created_at   REAL );  -- outcome tracking so skills can be trusted/retired by success rate CREATE TABLE skill_run (     id           INTEGER PRIMARY KEY AUTOINCREMENT,     skill_name   TEXT REFERENCES skill(name),     task_id      TEXT REFERENCES task(task_id),     outcome      TEXT NOT NULL,                     -- ok|failed|fell_back_to_llm     steps_taken  INTEGER,     created_at   REAL );  -- diagnosis log: what went wrong + what was decided (feeds future skills) CREATE TABLE diagnosis (     id           INTEGER PRIMARY KEY AUTOINCREMENT,     task_id      TEXT REFERENCES task(task_id),     trigger      TEXT,                              -- REPEAT|CYCLE|STALL|unexpected     cause        TEXT,                              -- selector_drift|auth_wall|...     chosen       TEXT,                              -- option taken     resolved     INTEGER,                           -- 1 if it unblocked     created_at   REAL ); ```  Useful learning queries:  ```sql -- skills worth trusting (deterministic path holding up) SELECT skill_name,        AVG(outcome='ok')              AS success_rate,        AVG(outcome='fell_back_to_llm') AS fallback_rate,        COUNT(*)                        AS runs FROM skill_run GROUP BY skill_name HAVING runs >= 5;  -- recurring stuck causes -> candidates for a new diagnostician rule or skill SELECT cause, COUNT(*) n, AVG(resolved) fix_rate FROM diagnosis GROUP BY cause ORDER BY n DESC; ```  ---  ## 5. Minimal end‑to‑end: anti‑loop + draft‑to‑skill pipeline  ```python # core/loop.py  — the ONE loop from core.workspace import open_draft, cleanup_draft from core.loopguard import LoopGuard from core.observation import build_observation from core.runner import run_step from core.registry import load_hooks, dispatch from core.journal import log_step from core.skillgen import extract_skill import time  class Ctx(dict): __getattr__ = dict.get; __setattr__ = dict.__setitem__  def run_task(goal, env, tools, stop, global_db):     draft = open_draft(goal)     hooks = load_hooks()     guard = LoopGuard()     ctx = Ctx(task_id=draft["task_id"], goal=goal, plan=None, redirect=None,               next_strategy=None, journal=draft["journal"], global_db=global_db)      dispatch("on_plan", ctx, hooks)                      # sequencer builds ctx.plan     obs = build_observation(env.page)      for idx in range(stop["max_steps"]):         # success check first (idempotent)         if success_met(stop["success"], obs):             dispatch("on_success", ctx, hooks)           # memory marks ok + win path             extract_skill(draft, global_db)              # draft -> SKILL.md, then cleanup             return "ok"          dispatch("before_step", ctx, hooks)              # sequencer may set ctx.redirect          # choose action (deterministic script first, LLM fallback inside run_step)         action = run_step(ctx, obs, bus=_Bus(ctx, hooks))         if ctx.redirect:                                 # sequencer veto             action, ctx.redirect = ctx.redirect, None          # ---- ANTI-LOOP GATE ----         stuck = guard.check(action["tool"], action, obs.hash())         if stuck:             ctx.stuck_reason = stuck             dispatch("on_stuck", ctx, hooks)             # diagnostician sets next_strategy             if ctx.next_strategy == "escalate":                 env.delegate_subagent(goal, ctx)         # subagent delegation             elif ctx.next_strategy == "ask_human":                 return "blocked"             continue                                     # never execute the tripped action          # execute         t0 = time.time()         try:             result = env.execute(action)             status, err = "ok", None         except Exception as e:             status, err = "error", str(e)         obs = build_observation(env.page)          ctx.last_step = dict(idx=idx, tool=action["tool"], args=action,                              args_hash=guard.hashes[-1] if guard.hashes else None,                              obs_hash=obs.hash(), status=status, error=err,                              latency_ms=int((time.time()-t0)*1000),                              subgoal=ctx.get("current_subgoal"))         dispatch("after_step", ctx, hooks)               # memory persists the step      dispatch("on_failure", ctx, hooks)                   # memory marks failed     cleanup_draft(draft, keep_artifacts_to=None)         # or keep work/ for postmortem     return "failed"   class _Bus:                                              # tiny event bus used by runner     def __init__(self, ctx, hooks): self.ctx, self.hooks = ctx, hooks     def emit(self, ev, *_a, **kw):         self.ctx.update(kw); dispatch(ev, self.ctx, self.hooks) ```  ### What happens on a first‑time site vs. the 2nd time  - *First time* (`web_register` doesn't exist): `run_step` has no script → LLM drives. Every step is journaled by `memory`. LoopGuard prevents re‑submitting the same broken form; on a trip, `diagnostician` re‑resolves the field by label or escalates. On `on_success`, `skillgen` keeps the `status='ok'` path, generalizes email/password/url into `{{params}}`, writes `SKILL.md` + `script.py`, validates by dry‑run replay, promotes into `skills/`, registers it, and *deletes the draft folder*. - *Second time* (same goal class): registry matches `web_register` by preconditions → `runner` executes `script.py` deterministically at ~0 LLM cost. The model is touched only if `UnexpectedState` fires (e.g., the site added a captcha), which logs a `diagnosis` row and may spawn a v2 of the skill.  ---  ## 6. Why this satisfies each requirement  - *No repeated actions:* `args_hash` ring buffer + cycle/stall detection gate every action *before* execution; a trip can never run the same action again — it routes to diagnosis. - *Explicit stop conditions:* declarative `max_steps` + `success` predicate list per task/skill, evaluated each iteration. - *Compact observation:* ~40 interactive elements with current values + a state hash, never raw DOM. - *Deterministic + LLM fallback:* `runner` runs the skill script first; LLM only on `UnexpectedState`. - *Self‑learning + draft cleanup:* `open_draft`/`extract_skill`/`cleanup_draft` give an auto‑created, auto‑destroyed temp workspace with its own journal that gets distilled into a durable skill. - *Clean multi‑layer composition:* three meta‑skills are flat, priority‑ordered hooks on a single loop's events — no skill ever nests or calls another, so there is no "nested skill hell."  _That's the complete document.
