## Key Findings

### High Priority (P0)
1. **Retry-wrapper** on all DB writes (3 attempts, backoff)
2. **migrate_schema.py** — idempotent migrations with checksums, `schema_version` table, rollback
3. **Migration 002** — Foreign Keys + `ON DELETE CASCADE` + indexes
4. **WAL mode** — `PRAGMA journal_mode=WAL`, `synchronous=NORMAL`, `busy_timeout=5000`

### Medium Priority (P1)
5. **rollback_turn.py** — revert N turns, cascade on decisions/artifacts, block on compacted sessions
6. **Memory.md optimization** — store only decision IDs (D3, D7), not full text. Saves ~1300 chars
7. **resume_context.py** — use `message_count` for skip, `token_count` for stop-condition

### Low Priority (P2-P4)
8. **compact_parents.py** — two-level compression: soft (every 10 sessions → summary), hard (>2000 messages → delete old)
9. **validate_last_turn.py --repair** — backfill from journal.log on gaps
10. **backup_db.py** — timestamped archive .db + .wal + .shm + journal.log + memory.md, retention policy

## Edge Cases & Mitigations

| Case | Mitigation |
|------|-----------|
| DB locked by other process | Retry loop 3 attempts, exponential backoff |
| journal.log truncated mid-line | Defensive JSON parsing, skip malformed lines |
| memory.md truncated | Parse by FIELD: prefix, fallback to SQLite |
| Parent chain too long | Count cumulative token_count, truncate before overflow |
| Disk full during write | Pre-flight `df -h /`, graceful degradation |
